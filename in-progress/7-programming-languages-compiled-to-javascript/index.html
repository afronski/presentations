ECMAScript 6 Proxies

  • As in the Mr Tate's book we should have an "ugly child".
  • Opal is a cross-compiler from Ruby to JavaScript. It represents whole range of projects which only transforms language which is not designed to work with client-side to JavaScript.
  • We can say that is eases pain and lowers the bar for newcomers. This is of course, not the only advantage of such approach, but for more experienced front-end developers having such language is such a relief.
  • Why I called Opal an ugly child? Because in comparison to other languages it does not add any value to the client-side development.
  • What Opal brings to the table? It is a 'method_missing' mechanism. Fortunately, in near distant future we can natively implement such behavior with use of ECMAScript 6 proxies.
